指针的定义与意义

上代码！

此时 p只是拿到了i的地址 不信我们打印出来看看

p = &i; 此时，说，p 指向了 i

但是要区别，p是一个变量，可以指向其他的位置，而&i是一个常量，代表了变量存放的地址

%p的意思是 输出16进制的地址

那如何能得到我存放地址的那个变量的值呢？

间接运算符登场！

定义了一个指针可以指向所指向变量的类型！也就是说 int *p;中的 int * 用来接受 int类型变量 的地址的。你细品！！

在次回看前面的一通操作观察可以很快发现.

指针到底有什么用？为什么需要有奇奇怪怪的指针？

那么请你想一个问题，我现在需要让你设计一个函数，目的是互换两个变量的值，你会怎么做？

你肯定会这么做，但是有个问题，输出的结果还是 a = 5, b=10 并没有改变！

原因是在函数中的 a 和 b 不是main中的那个a和b，是新的变量。

所以你做的是无用功…..

但是函数还有个功能 return 对不对！ 很遗憾，return 只能return一个变量，不能是两个！！！

唯一能解决的方案就是指针！

可以总结出，我通过指针直接拿到了main函数里的 a 和 b 的值 ，并对它们进行了修改。

总结指针的几个操作

赋值

p1=0x7ffee0b51bd0
&p1=0x7ffee0b51bb0

地址不同对吧，原因是p1的意思是数组的第一个元素的地址（也就是这个数组的地址）

而&p1则是指p1这个指针变量的地址，注意 ： 指针变量也是变量，它同样也需要内存地址。并且，在&p1=0x7ffee0b51bb0下储存的内容正是0x7ffee0b51bd0（就是数组的地址data）

解引用

p1 实际上拿到的是data 数组首个元素的地址，我们需要 * 运算符，取到该地址上存储的值。

指针和多维数组

这里你可以想象成一个4*2的矩阵，或者有4大盒子，每个大盒子里含有2个小盒子。

如果直接打印arr的话

arr 是一个包含2个int值数组的地址

为什么是两个？我们来 +1 看看

arr+1 和 arr 来进行比较 0x7ffeefbff578 0x7ffeefbff570 我的系统int 是4个字节，观察后两位差8个，说明一个arr内含2个int类型的地址

*arr+1代表什么？与 arr 有何区别？

可以惊喜的发现，两个地址竟然相同！

好吧，你可以理解为arr是个大盒子，里面套了两个小盒子,分别是*arr 和 *arr+1 。

那么当然 arr 与 *arr 地址肯定相同,因为起始地址相同。换一个角度，arr包含 *arr 。

arr[0] 表示了每个大盒子里的第一个小盒子的地址，故arr[0]+1 第一个大盒里第二个小盒子的地址

重点来了！

arr 仅仅能代表第一个第一个大盒子的地址并且 == 第一个大盒子里面第一个小盒子的*地址

所以我们需要取地址的地址！

分解来看看

  1. arr = 0x7ffee04e5ba0 取到第一个大盒子内第一个小盒子*的地址

  2. **arr=2 取到第一个大盒子内第一个小盒子地址上的值

    const与指针

让p3指向其他变量，编译器报错！说明不能改变的是p3 与 i 的关系。也就是不能再让p3再指向其他变量了！

不能操作的是：不能通过 *p1 来修改值。其他操作都可！

一些其他情况

那么分别表示什么意思？

其实只有两种情况，要看count 在 * 前还是在 * 后

  • count 在 * 前 —–> 不能通过指针来修改变量
  • count 在 * 后 —–> 所指的东西不能再改变了
