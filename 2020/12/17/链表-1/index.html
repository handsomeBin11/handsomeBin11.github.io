头指针和头节点的区别与联系

先上图吧！

  • 头指针

    1. 其实就是指向链表第一个节点的指针，它用来记录链表的位置，因为你懂得，链表之所以为链表就是每个节点之间是有联系就像一条链子一样环环相扣。这个指针会告诉你第一个节点的位置在哪，然后你会找到第二个节点，第二个节点会告诉你第三个节点在哪…..以此类推。

    2. 无论链表是否为空，头指针不能为空。因为它具有标示意义。

    3. ❕注意⚠️！！！头指针不能为空

  • 头节点

    1. 头节点的作用是为了统一方便而设立的，由上图能看出置于第一个节点之前，但是它其实并没有什么意义！
    2. 有的同学会好奇，那头节点存在有什么意义吗？其实它的作用就是为了方便之后插入和删除的时候和其他节点的处理方法统一。有了它更方便，没有也可以！

链表的表示形式

通常有两个内容

  1. 存放该节点的值
  2. 存下一节点指针的地址

链表的读取

算法思路：

                1. 先声明一个p（屁，哈哈哈）用来记录一个节点
                2. 设定一个计数器（计数器当然从一开始记，只有程序员才从0开始！）
                3. 当第一个节点存在**并且**j还没有累加到i的时候循环继续
                4. 如果p不存在并且j大于了i说明超范围了，报错
                5. 把当前p（也就是走到要查的节点哪）的值给了e
                6. 成功！

链表的插入

不像之前的线性表一样，（在排队过程中）中间要有人插队的话，从这人之后的每一个人都有后退一个位置，这样效率很低！但是链表的优势就粗来了，它不用惊动那么多的节点，只把左右俩个节点搞定就行。就像这样

注意这里的顺序，是先p->next = s->next 然后 p->next = s 因为如果顺序反了就相当于 s->next = s

a i+1没了上级！链子断了！！！！

算法思路：

  1. 定义节p节点和s节点。（s是新的节点）并让p拿到链表的地址
  2. 计数器 j = 1
  3. 往后寻找要插入的i的位置，当p存在的时候 j 还没有到i的时候循环继续 p = p->next
  4. 如果p不存在或者超范围return错误
  5. 用malloc生新节点给了s
  6. 把节点的值e给了p的data
  7. 接上去
  8. 返回ok

删除节点

其实你可以想象就是三个人好基友/好闺蜜手拉手一起溜达，但是途中中间这位大哥突然想上厕所了，所以我们只需要（绕过他）将两边的人手拉手🧑‍🤝‍🧑即可。

算法思路：

  1. 创两个变量 p q
  2. 计数器 =1
  3. p拿到链表的地址
  4. 当p的下一个存在并且j没到i的时候，p一直往下走，计数器++
  5. 如果p的下一个不存在或者j>i了报错
  6. 开始赋值即可
  7. 记得将返回的结果给了*e
  8. 释放q

尾插法

就是每次将新节点插入到队尾，这样才符合我们日常的逻辑，新来的站在后面。

有人肯定会心生 “卧槽”，那个 r->next = p;和 r = p; 是什么鬼！

我们来画个图解决！

此刻要注意，L 于 r 的关系，r只是指向链表的尾部，r会随着链表的增加而变化（始终指向尾部）

而L是整个链表，是会随着循环日益增长的节点的链表。

r -> next = p的意思就是将尾节点交连接上新的节点。
