数据库查询总结

数据库对于我来说很难受，或许是我天生对表格不敏感吧。。。导致我学起来很是费劲，不过whatever!让我们来总结几个常用语句吧！

在使用某库的时候别忘了要use 库名哦～

基础查询

语法：select 查询列表 from 表名

特点是：

  1. 查询列表可以是：表中的字段，常量值，表达式，函数
  2. 查询的结果是一个虚拟的表格
查询表中的某个字段

SELECT 字段1，字段2，字段n from 表名

查询所以字段

SELECT * from 表名

起别名

Select last_name 姓,first_name 名 from 表

Select last_name as 姓,first_name as 名 from 表

去重 distinct

例如：查询员工表中涉及到的所有的部门编号

很多员工来自相同部门所以去除重复, distinct要放在select后字段前

Select distinct department_id FROM employees;

连接字段

案例：查询员工名和姓连接成一个字段，并显示为 姓名

Select concat(last_name,first_name) from 表名

条件查询

语法：

分类：

  1. 按条件表达式筛选：> < = != <> >= <=

  2. 按逻辑表达式：用于连接条件表达式 ：&& || !
    and or not

  3. 模糊查询： like between and in is null

    案例

    案例1: 查询工资z在10000到20000之间的员工名、工资以及奖金

    模糊查询

    like 一般是和通配符一起使用，所谓通配符有两种

    第一种: % 可以多个字符，包含0个字符

    第二种 _ 单个字符

    案例

    查询员工名中包含字符a的员工信息

    分析：注意问题中写到’的员工信息’，暗示所有员工的所有字段所以 是 * 号

    查询员工名中第三个字符为e，第五个字符为a的员工名和工资

    分析： ‘的员工名和工资’

    查询员工名中第二个字符为_的员工名

    分析：’的员工名‘ ，如果我们写筛选条件是 _ _ %的时候编译器无法识别是第二个，所以要用到‘分割符’—>$

    between and

    ①使用between and 可以提高语句的简洁度
    ②包含临界值
    ③两个临界值不要调换顺序

    查询员工编号在100到120之间的员工信息

    如果我们要实现的是 ： 查询员工的工种编号是 IT_PROG、AD_VP、AD_PRES中的一个员工名和工种编号

    这样写代码过于孬种不简洁，于是我们引入了 In 字句 来表示某字段的值是否属于in列表中的某一项。

    Is null 用is null 或者 is not null 判断某些字段是否是空值

    案例1：查询没有奖金的员工名和奖金率

    案例1：查询有奖金的员工名和奖金率

    查询没有奖金的员工名和奖金率(安全等于)

    分组查询

    案例一：查询查询各job_id的员工工资的最大值，最小值，平均值，总和，并按job_id升序

    案例#2.查询员工最高工资和最低工资的差距（DIFFERENCE）

    3.查询各个管理者手下员工的最低工资，其中最低工资不能低于6000，没有管理者的员工不计算在内

    group by 与 from 之间用where子句， 分组后, 添加筛选条件用的是having 字句

    5.选择具有各个job_id的员工人数

    分组查询的格式

    select 查询列表
    from 表
    【where 筛选条件】
    group by 分组的字段
    【order by 排序的字段】;

    特点：
    1、和分组函数一同查询的字段必须是group by后出现的字段
    2、筛选分为两类：分组前筛选和分组后筛选

    如果能分组前筛选尽量分组前筛选，能提高效率。

    案例：查询哪个部门的员工个数>5

    案例：每个工种有奖金的员工的最高工资>6000的工种编号和最高工资,按最高工资升序

    案例：查询每个工种每个部门的最低工资,并按最低工资降序

    注意：group by 可以对对个字段进行分组。
