当队列为空的时候

front 和 rear都指向头结点

队列的结构定义

入队操作

  • 新弄一个结点
  • 装配好
  • 接到队尾
  • 刷新队尾的指针

出队操作

  • 判断是不是空的
  • 新建一个指针p指向要删除的元素
  • 把值交出来
  • p拿到front 的next

队列的顺序存储

Posted on 2021-11-14 Edited on 2024-01-24 In 数据结构与算法

基本概述

队列的顺序存储结构其实也是基于线性表来完成的，只不过是受到限制的。它的限制是指能在一端删除元素。另一端添加元素。删除的元素的那一端成为队头，能添加元素的一端为队尾。

当队列为空时

rear == front 都指向0号位置

出队时

只需要移动front 指针和 rear即可，不用像线性表的线性存储一样一个一个往前挪。

这样做导致的问题

当front不断出队的时候指向最后一个元素，此时rear已经没有办法加入元素，而前面剩下3个位置没有办法用，这样会导致假溢出。

如何解决

我们可以把表前后链接成一个环，利用算法让rear回到前面。

判断队列满的条件就是 (rear + 1)%MAXSIZE == front ，必须空一个出来才能和队空的条件区分开。

计算队列共有多少个元素

队列的结构定义

  • 需要一个data数组
  • int rear
  • int front

入队

  • 先判断满没满
  • 在队尾赋值
  • 队尾进行算法调整

出队

  • 判断是不是空
  • 队头的元素拿出来
  • 刷新front

为何要用尾指针？

1. 时间效率问题

即便是到了循环链表，从任意结点出发能都能访问到整个链表。但是还有一点不是很perfect！那就是

  • 当访问第一个结点是O(1）
  • 当访问最后结点却是O(n)

那怎么能修改呢？可以用尾指针！

  • 用尾指针指向最后一个结点

    • 当访问第一个结点显然是 rear -> next -> next

    • 当上面👆这样改之后，当访问最后一个结点的时候其时间复杂度尾变成了 O(1)，比之前的来的高效的多！
