235. 二叉搜索树的最近公共祖先

发表于 2022-01-25 更新于 2024-01-24 分类于 Leetcode树专题

题目描述

给定一个==二叉搜索树==, 找到该树中两个指定节点的最近公共祖先。

百度百科中最近公共祖先的定义为：“对于有根树 T 的两个结点 p、q，最近公共祖先表示为一个结点 x，满足 x 是 p、q 的祖先且 x 的深度尽可能大（一个节点也可以是它自己的祖先）。”

例如，给定如下二叉搜索树: root = [6,2,8,0,4,7,9,null,null,3,5]

示例 1:

输入: root = [6,2,8,0,4,7,9,null,null,3,5], p = 2, q = 8
输出: 6
解释: 节点 2 和节点 8 的最近公共祖先是 6。
示例 2:

输入: root = [6,2,8,0,4,7,9,null,null,3,5], p = 2, q = 4
输出: 2
解释: 节点 2 和节点 4 的最近公共祖先是 2, 因为根据定义最近公共祖先节点可以为节点本身。

说明:

所有节点的值都是唯一的。
p、q 为不同节点且均存在于给定的二叉搜索树中。

思考

😅一开始没有看到是二叉搜索树！走了很多弯路。。。。。这道题应该用到搜索树的性质来做。

我们画张图容易理解！

  • 如果判断两值都小于当前节点往左走
  • 再如果判断两值都大于当前节点往右走
  • 否则当前节点就是要找的节点。再break出去。

12. 路径总和

发表于 2021-12-10 更新于 2024-01-24 分类于 Leetcode树专题

给你二叉树的根节点 root 和一个表示目标和的整数 targetSum 。判断该树中是否存在 根节点到叶子节点 的路径，这条路径上所有节点值相加等于目标和 targetSum 。如果存在，返回 true ；否则，返回 false 。

叶子节点 是指没有子节点的节点。

示例 1：

输入：root = [5,4,8,11,null,13,4,7,2,null,null,null,1], targetSum = 22
输出：true
解释：等于目标和的根节点到叶节点路径如上图所示。
示例 2：

输入：root = [1,2,3], targetSum = 5
输出：false
解释：树中存在两条根节点到叶子节点的路径：
(1 –> 2): 和为 3
(1 –> 3): 和为 4
不存在 sum = 5 的根节点到叶子节点的路径。
示例 3：

输入：root = [], targetSum = 0
输出：false
解释：由于树是空的，所以不存在根节点到叶子节点的路径。

提示：

树中节点的数目在范围 [0, 5000] 内
-1000 <= Node.val <= 1000
-1000 <= targetSum <= 1000

链接：https://leetcode-cn.com/problems/path-sum

思路

题目的意思就说，给你一个路径sum，从根到叶子让你看看有没有那条路经上的结点加起来=给的sum

所以我的大体思路就是，前序遍历也就是DFS遍历深度遍历整课树，如果遇到一个结点就从sum 里面扣除当前结点的值。如果当发现走到叶子结点的时候当前的sum恰好为0的话就是说明符合题目所给的路径。

  • 检测当前root节点是不是空
  • 检测当前是不是叶子结点（所有子树为NULL），如果是的话，看看是否是0
  • 深度遍历左树并且-当前值 || 深度遍历右树-当前值

示例 1：

输入：root = [3,9,20,null,null,15,7]
输出：2
示例 2：

输入：root = [2,null,3,null,4,null,5,null,6]
输出：5

提示：

树中节点数的范围在 [0, 105] 内
-1000 <= Node.val <= 1000

思路

根据题目中所给的，最小深度是从根节点到最近叶子节点的最短路径上的节点数量。

这里注意加粗的结点数量，假设左树有一个分支且分就一个结点，你这里需要把结点数+1，因为有根结点

  • 先判断根的结点是不是空 —-> 如果是的话就return 0
  • 再判断该根下的左右子树是否同时为空 —> 同时空就返回 1代表最小深度为1
  • 如果左树存在 则递归遍历左树 并且刷新最小值
  • 如果右树存在 则递归遍历右树 并且刷新最小值
  • 返回最小值+1
