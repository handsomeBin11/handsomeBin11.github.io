指针和多维数组

发表于 2021-04-22 更新于 2024-01-24 分类于 C语言理论部分

先看一段代码👇

zippo = 0x7ffee5fc3b50
&zippo[0] = 0x7ffee5fc3b50
&zippo[0][0] = 0x7ffee5fc3b50

惊奇的发现地址一样！

  • 根据数组的知识，Zippo 数组名就是该数组的首地址。

  • 根据上图可见，&zippo[0] （蓝色）其实是一个含有两个元素的小数组。那么根据我们的理论，既然是数组，它的表示方法就是首个元素的地址，它的地址是那个（黄色）那个位置元素的地址。
  • zippo 是整个二维数组的名，它的地址又恰好与那个小的一维数组zippo[0] 地址重合
  • 自然，那个黄色框框里的(也就是存放2那个元素)位置的地址为&zippo[0][0]

👌，那我们最终🉑️推出，zippo = &zippo[0] = &zippo[0][0] = 0x7ffee83eab50

—-> 进一步推理

  • 对于整个二维数组zippo来讲，他含有4个 1维的数组，每个一维的数组里面又含有2个元素。所以对于zippo的第一个元素就表示占用了两个大小的 int 对象的地址。
  • 而对于zippo[0] 来讲，他只是一个占有一个元素的的地址。

​ zippo = 0x7ffee83eab50 , zippo + 1 = 0x7ffee83eab58
zippo[0] = 0x7ffee83eab50 , zippo[0] + 1 = 0x7ffee83eab54

从👆那个输出结果可以看出，当我们+1 之后，zippo 和 zippo[0]的结果是截然不同的,但是开始位置是相同的。

  • zippo + 1 一下跳了两个int —-> 8. 说明 zippo 是占有两个int对象的地址
  • zippo[0] +1 一下跳了一个int —-> 4 说明 zippo[0] 是占有一个int对象的地址。

*zippo = 0x7ffee9c4fb50 ,

&zippo[0][0]= 0x7ffee9c4fb50

​ 因为*zippo 代表取了一层地址，相当于是里面小数组的第一个地址也就是 zippo[0][0]（首地址）

*(zippo[0])= 2 , &zippo[0][0]= 2

  • 因为zippo[0]代表着是小数组里面的首个元素的地址，* 它 就是取它地址上的值，那当然是 2啦。

————->所以说，如果我们想通过指针取得地址上的值我们需要👇

*(zippo[0])= 2 , zippo[0][0]= 2, **zippo = 2

  • 需要取两层 * ,
    • 第一层 *代表小数组的第一个元素的地址
    • 再取一层 * 代表小数组里面的第一个位置上存放的值。

结构体

发表于 2021-04-18 更新于 2024-01-24 分类于 C语言理论部分

为蛇么有结构体？

如果你想连续存放一些数据，你可以用数组，但前提条件是数组必须每个元素的值都是相同类型的

但如果你想存放一些不同类型的数据，把它们弄成一个整体—-出现了结构体。

声明在函数外还是内？

  • 如果你想只在某个函数里使用，就定义在函数里，就像上面的main函数里一样。

  • 如果你想在程序的每一处都能够使用，那么你就定义在外面如下：

声明结构的形式–3种形式

  • struct point{
  int x;
  int y;
};

struct point p1,p2;


  • struct point{
  int x;
  int y;
}p1,p2;
//相当于说，定义的时候顺便定义了 p1 p2
//以后你想用该结构变量的时候，还是可以采用 struct point *** 的形式来定义的

如图，当你花括号里这样定义的时候，编译器会自动帮你一一对应赋值结构中的变量。

如果你想指定的几个赋值的话！采用 .神马 的方式来进行赋值

当然，你没有赋值的变量会像数组那样，自动赋 0 值

访问结构中的成员

结构和数组的情况有点像

  • 数组用下标的方式访问里面的成员

    • arr[ 3 ] = 10;

  • 结构 用 . 运算符 和 名字来访问其成员

    • today.day
    • student.Name
    • p1.x
    • p1.y

结构运算

  • 要访问整个结构，直接用结构变量的名字

  • 结构可以做

    • 赋值

    • 取地址

    • 传递给函数的参数

    • p1 = (struct point){5,10};//强制转换
//相当于 p1.x = 5 p1.y = 10

    结构指针

    注意⚠️，结构的名称不是变量的地址（这里和数组是不同的），所以说加要取地址运算符。

结构与函数

  • 整个结构是可以作为参数传入函数的
  • 这时候，是在函数内新建了一个新的结构变量（克隆体），然后把调用者的值复制进去
  • 也可以返回一个结构
  • 这与数组完全不同

字符数组

注意这里不是C语言的字符串，以为不能用字符串的方式做计算。

真正的字符串

注意⚠️ ：可以看出在最后的位置上有 ‘\0’ 这才叫真正的字符串，因为只有这样，我们才能修改字符串里面的值。

字符串的特点是什么？

  • 以 整数0 或者 ‘\0’结尾，（注意这里是 ‘\0’) ，不是 **字符’0’**。
  • 那个 0 意味着是字符串的结束标志，它不属于字符串，不是字符串的一部分。
    • 也就是计算字长度的时候不包括 0
  • 字符串是以数组的形式存在，以数组或者指针的形式访问。
    • 更多的是用指针代替
  • string.h 里有许多的字符串函数。

字符串变量

  • char *str = “hello”;
  • char word[] = “hello”;
  • char line[10] = “hello”; —> 这里也就意味着我们申请了10个位置，但其实这个字符占据的是6个字节的位置，也就是还需要算上末尾’\0’。 也就是编译器会替你生成结尾的那个0的。

总结一下 究竟什么是字符串

  • C 语言的字符串是以字符数组的形式存在的

    • 不能用运算符对字符串做处理运算。（因为c是1970年代的语言，当时还不需要处理字符数据）
    • 通过数组的方式可以遍历字符串

  • 唯一特殊的地方在于字符串的字面量可以用来初始化字符数组，这也是c语言‘最懂字符串’的地方了，因为它懂双引号是表示字符串。
