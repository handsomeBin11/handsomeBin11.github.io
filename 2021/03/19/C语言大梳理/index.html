& 取地址所得那个地址，交给一个整数变量来接收显然是不合理的。

得有一个变量来接收取地址所得到的地址。那么究竟什么样的类型能接收我们取地址所得到的那个地址呢？

指针类型的变量

但我要注意⚠️ ！！！：

int * p 的意思是说： — > * p这个变量是一个int类型的指针变量 也就是说，我能够接收int类型的变量的地址。

我们是把 *** 号给了p 而不是给了 int**

指针变量和别的变量的区别

指针变量里面存放的只能是别的变量的地址。

而其他变量里面放的都是值。

👆 上面两种情况，相当于说 只定义了一个 int *p的指针变量 ，而那个q只是一个普通的 int 类型。

定义的时候究竟发生了什么

上面， 这行代码究竟发生了什么？看下图

  1. 定一个int 类型的 i = 6； i变量的地址为0x2000
  2. 定义了一个int类型的指针p ，它接收了i的地址 ，上图，i的地址为0x2000 那么p变量里面保存的就是0x2000（也就是i的地址）
  3. 此时我们说，p变量指向了i变量。

作为参数的指针

其实，当main函数里的 f(&i) 我们把i的地址拿出来传给了f函数 f( int *p ) 黄色箭头

p 变量保存了那个i的地址（绿色），所以蓝色箭头f 函数里的 p 指向了main函数的 i 。 也就是说现在f函数里的p拿到了main函数里的i的访问权限。但是，p并不知道那个 i 叫 i ，只是拿到地址。

g函数只是拿到了i的那个值，但是和外面的那个i 没有半毛钱关系。

访问那个地址上的变量

"   *   " 号**单目**运算符能够访问道拿到变量地址上的**那个值**。

当 p 拿到了地址之后能做什么？当然是读和写喽。

*p 就拿到了0x7ffee057ab7c上存放的值也就是 = 6

来看看究竟发生了什么，我们刚刚用 p 拿到了i的那个地址，我们说p指向了外面的那个i，也就是拥有了访问i的权利。

当我们用 单目运算符 * 的时候。*p就意味着，先通过p拿到的地址 顺藤摸瓜2⃣️ 通过地址访问到那个i的位置 ，在通过 * 号拿到3⃣️ i里面的地址上的那个值，即值6。

此时 我们可以把 *p当作一个整体，这个整体就是一个int类型的。

一点点🤏小修改看看会发生什么

  • 当我们在f函数里面修改了*p的值的时候，也就意味这修改了那个i的值。

  • 我们仍然说，函数的调用的时候是值得传递，但是因为传递是地址的值，所说就 p代表i的地址，*p也就能代表 整个 i

  • 修改*p 的时候 实际上就是修改了 i 的值

  • 当我们 调用 g函数的值时候，可以发现i的值已经被改变了，也就印证了上面的结论。

指针的运算符 * & 以及护反作用

  • *& p -> *(&p) -> *(p的那个地址) -> 得到地址上的变量 -> p

  • &*ptr -> &( *ptr) -> &(p) -> 也就是原来那个指针p
